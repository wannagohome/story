# QA Journal — 세션 간 기억 저장소

> **이 문서의 목적**: QA를 수행하는 에이전트(사람/AI)가 매 세션 시작 시 반드시 읽어야 하는 living document.
> 세션이 끝나면 발견한 사항을 여기에 기록하여, 다음 세션이 같은 실수를 반복하지 않도록 한다.
>
> **규칙**: 새 항목 추가 시 날짜와 세션 식별자를 명시할 것. 해결된 이슈는 삭제하지 말고 ~~취소선~~ 처리할 것.

## 5. 교훈 (Lessons Learned)

"이렇게 하면 안 된다", "이렇게 해야 한다"를 구체적으로 기록한다. 같은 삽질을 두 번 하지 않기 위해 존재한다.

<!-- 예시 형식:
- **[2026-03-05] L4 subagent 프롬프트에 방 코드를 빠뜨리면 안 된다**
  증상: 에이전트가 새 방을 만들어 각자 혼자 플레이함.
  해결: 프롬프트 첫 줄에 `방 코드: XXXX` 명시 필수.

- **[2026-03-06] GM Eval은 트랜스크립트가 최소 10턴 이상이어야 의미 있다**
  증상: 3턴짜리 게임으로 GM 평가 시 "정보 부족"으로 모든 항목 3점.
  해결: GM Eval 입력 최소 조건을 10턴 이상으로 설정.

## 작성 가이드

### 새 항목 추가 시 규칙

1. **날짜 필수**: `[YYYY-MM-DD]` 형식
2. **세션 식별자 권장**: 어떤 세션에서 발견했는지 (예: `ses_abc123`)
3. **구체적으로**: "잘 안 됐음" ✗ → "fixture AI 모드에서 /talk 명령어의 응답이 빈 문자열로 옴" ✓
4. **해결 방법 포함**: 문제만 적지 말고, 해결했다면 어떻게 해결했는지도 기록
5. **삭제 금지**: 해결된 이슈는 ~~취소선~~ 또는 상태를 `fixed`로 변경. 히스토리 보존

### 섹션별 용도

| 섹션 | 언제 쓰는가 | 예시 |
|------|------------|------|
| **항상 기억할 것** | 모든 세션에 영향을 주는 보편적 규칙 발견 시 | "fixture AI에서는 API 키 불필요" |
| **알려진 이슈** | 버그 발견 시 | "/look이 위치 설명을 빠뜨림" |
| **환경 메모** | 환경 설정에서 삽질한 후 | "Node 18에서 안 됨" |
| **실행 기록** | 테스트 세션 종료 시 | "L1 세션관리 12개 PASS" |
| **교훈** | 실수를 하고 해결한 후 | "프롬프트에 방 코드 필수" |
| **결정 로그** | 여러 선택지 중 하나를 고른 후 | "L1은 fixture AI만 사용" |
