# NFR: Non-Functional Requirements (비기능 요구사항)

**Traces**: NFR-001 ~ NFR-029 (PRD 기준), QA-NFR-101 ~ QA-NFR-106 (QA 추가)
**Layer**: L1 + L2 + L3 + L4 (전 레이어 임베딩)
**인터랙션**: Protocol level + TUI + CI matrix
**AI 사용**: fixture AI + real AI 분리 적용

---

## AI 모드 분리 규칙

- `Fixture 가능`: 프로토콜/보안/호환성/구조 검증처럼 결정론적 검증이 가능한 NFR.
- `Real AI Only - L3/L4`: 실제 LLM 지연/행동 품질이 직접 지표에 영향을 주는 NFR.
- 본 문서에서 **NFR-001은 반드시 Real AI로 게이트**하며, fixture 결과는 회귀 기준으로만 사용.
- 향후 서사 일관성/캐릭터 일관성 같은 AI 품질 NFR이 추가되면 fixture 검증 금지, `Real AI Only - L3/L4`로만 운영.
